Why PowerBrowser?

A thought of making a browser comes with ambition, not need. Throughout the world, there are many different browsers available. Making this type of project only comes to mind when making something interesting, or fun, creative or ambitious. This product has the reasoning of ambition and creativity, not a need. However, an idea of this kind is no easy job, hence making this is a special occaision. The Global Context of this project is Scientific and Technological Innovation focusing on the creation process and the capability of PowerBrowser as a whole project.

Why Python?

For starters, my choice of a base programming language is Python. This is because since about 3rd-4th grade, I had been learning this language. The base and popularity was also enormus, from anyone I would hear or see talking about programming, mentioning Python was a must, because of it's logic and simplicty, usability, achievements, capabilities and what not? This was the only reason I had to select Python as my base programming language to program this in.

Why PyQt6?

Even though I know what my base programming language for this project is, I still would've needed a library to work on it. A Library is a setup and type of add-on code to Python which is used as a secondary base to writing applications out. With this in mind, after research PyQt6 was the idea I had developed in mind. PyQt is a GUI-development library built by Riverbank Computing, which provides many hundreds of built-in widgets, including a fully functional web engine based on Chromium. This research was a perfect answer to my problem as I had the knowledge of PyQt5 before having the idea of this project. After research of comparisons of the two version, I found out that the difference between the two was not much, and was not too much to cope to. With the conclusion in mind, I started out making trial applications just to see how the library works like. In the end, after testing this model out, I had fully made up my mind to use PyQt6 as my base library for this particular project.

Initial Idea

Originally, even though I had figured out all necessary information for programming this browser, I still insisted to try to make the browser with my far stronger knowledge on PyQt5, just for seeing how it would look like. At first, my idea was to make one browser, and that would be about enough for making the project. Then after making the entire browser, I first published it on GitHub, and did self-evaluation with peer evaluation of the browser, from which I concluded that making this type of program would not at all be enough for making a good project. From the point onwards, I decided to completely scrap PyQt5 and use PyQt6 because it would have a more up to date version the the Chromium Framework. For a bit of knowledge, after researching, PyQt5 had Chromium Version 87 with Patch version of 90, whereas the latest version of PyQt6 has Chromium Version 134 with Patch Version 140, which clearly indicates that Qt6 has a far more advanced and up to date version of Chromium. When trying out Qt6, it overall even looked more organized and nicer compared to Qt5. From this point onwards, I decided to upgrade the browser until it reaches my limits to programming.

Evolving of PowerBrowser

As mentioned previously, the first version PowerBrowser achieved it's main goal but lacked polish in design, capability and a modern web engine. With this in mind, PowerBrowser 2 already had a huge pluspoint of using a more up to date version of Chromium. It only needed to be improvised upon. In PowerBrowser 2, the idea of making stylesheets was new for me. I already had the idea of how stylesheets work, but never knew that it could be implimented onto PyQt applications. Just for an idea, stylesheets are lines of code which help in the stylizing of something, it is only programmed in the language of CSS (Cascading Style Sheets). After some reasearch, I found out that PyQt heavily relies on HTML (HyperText Markup Language) and CSS, both web-oriented programming languages. With this in mind, I started to use stylesheets to put a heavy skin on PowerBrowser, making it look like a completely different product. For design, I also updated the Icons of the browser, which now made it look like a completely new product. The new useful features of the second edition of the browser which I added was the ability to view the history and the bookmarks of what the user has been up to. It would store the following data in a text file (.txt). To show the history and bookmarks, it would rely on an application like Notepad to view the data.

The next couple of versions all improvised the foundation of PowerBrowser v2. The third version did not add much to itself rather add adding a more heavier and stylized look to itself by using CSS stylesheets. The major improvement in this edition was the ability to show bookmarks and history without using other 3rd Party applications. It also had shortcuts in it's url bar, like "yt" directly loading "youtube.com". There also was a major improvement in the dark mode of the browser, looking clean and proper compared to the previous editions, where the colours would look slightly off and sizes of certain parts of the browser looking a bit funky.

The 4th version however, was a new era of the browser. It had continued on the same stylesheet of Version 3, but with a general themeing of green instead of Light Blue. The major improvement was the simplicity of the browser, there was no longer a menu bar and recommendations bar. The main navigations bar had now been moved to the top of the browser. Another major improvement was the adding of the new Settings panel. It had all menu features within itself. The history and bookmarks data would now also be showed directly on the settings panel. Another new major feature of this edition is the ability of mute the audio of a site, it is now present when right clicking the desired tab you would want to mute.

The 5th, 6th and 7th version of the browser continued on the themeing of the 4th edition but using the colour Light Blue again instead of Green. Now, the main features are that all user data (history bookmarks, downloads, etc...) data will be now stored in SQLite files, which are more organized and hard to tackle directly than text files. They also keep data organized more. Storing manual bookmarks in these edtions have also been removed, as now whenever you need to make a bookmark, it makes a direct bookmark of the site and stores it in SQLite, to trigger this, you can right-click on the tab and select the option of adding the bookmark, or you can use Ctrl+D to make the bookmark, both work well. In versions 6 and 7, there is a new download manager storing all download history, and a direct searching feature, searching something with Google, or opening something in Britanica, Wikipedia, as PDF, etc...

A notable improvement in the the 7th edition is that now History and Bookmarks data will be directly shown of the database rather than showing it as a list of text. this makes the UI look cleaner and more efficient. For extras, all editions now show their specification when pressing "About" in the Settings Tab.

Testing Process of the Browser

Making a product like this requires tremendous ammount of skill, patience and testing, without testing, this application is completely impossible to make. The testing process has been done manually (by hand) after every single feature was added, which was the safest way of making sure the application works to it's most. The error handling was also not of the easiest, as sometimes the errors were too much to cope with, to which I had to completely re-write and research the internet to make sure my programming was correct. The making of the cascading stylesheets for design also required tremendous ammount of trial and error. At first, you really have to settle your eyes over what you are seeing, which is the hardest part of all, because picturing the entire application is simply impossible, if you are able to overcome the following thoughts at first, most likely that same ugly toolbar fits perfectly with the rest of the UI (User Interface) perfectly and looks neat and clean.

Conclusion

Overall, this project has been no easy job to create and make. I have took help and reasearch from multiple sources and debugged heaps of errors throughout the history of the project. I am really happy to pull off all challenges in a somewhat sucessfull manner and I really hope to imrpove this project even more in the future. For now I don't have any plans on making this project better, but I will find a way to make it better. Thank You for spending your time reading this documentation. I really appreciate it.
